# tmux calendar chip

Peach-colored status chip showing the next event from Google Calendar.

```
[ EBM Fusion Steering [NEW] @ 15:15]
```

## Architecture

| File | Role |
|---|---|
| `fetch.sh` | Calls `gcalcli`, writes today's events to `~/.cache/tmux-calendar/events.json`, signals tmux refresh. Fires on every new tmux session via `session-created` hook. Self-throttles to once per 60s. |
| `chip.sh` | Reads the JSON cache, prints the next or current event. Called every status-interval tick (5s — local file read only, no network). Triggers a background refresh of `fetch.sh` if the cache is >15min old. |
| `calendar.conf` | Defines the catppuccin status module. Sourced from `tmux.conf` after TPM loads. |

Wired into `tmux/tmux.conf`:
- `session-created` hook → `fetch.sh`
- `source ~/.config/tmux/calendar/calendar.conf` (after `run '...tpm'`)
- `status-right` includes `#{E:@catppuccin_status_calendar}`

## One-time setup (per machine)

1. Install dependencies:
   ```bash
   brew install gcalcli jq
   ```

2. Set up a Google Cloud OAuth client for `gcalcli`:
   - Go to https://console.cloud.google.com → create a project
   - Enable the **Google Calendar API**
   - Create credentials → **OAuth 2.0 Client ID** → **Desktop app**
   - Note the client ID and client secret

3. Authorize gcalcli:
   ```bash
   gcalcli init
   ```
   Paste client ID + secret when prompted, then authorize in browser.
   Verify with `gcalcli list` — should show your calendars without prompting.

4. Create the machine-specific config file (not in the repo):
   ```bash
   mkdir -p ~/.cache/tmux-calendar
   echo "CALENDAR_EMAIL=your.email@example.com" > ~/.cache/tmux-calendar/config
   ```
   This filters `gcalcli` to your primary calendar. Without it, `gcalcli` returns
   events from every calendar you're subscribed to (including shared team
   calendars), which is rarely what you want.

5. Reload tmux: `prefix r` (or restart). Chip should appear within 60s.

## Files outside the repo

- `~/.cache/tmux-calendar/config` — `CALENDAR_EMAIL=…`
- `~/.cache/tmux-calendar/events.json` — JSON cache, regenerated by `fetch.sh`
- `~/.cache/tmux-calendar/fetch.log` — last fetch's stdout/stderr
- `~/.config/gcalcli/oauth` — OAuth token (created by `gcalcli init`)

## Manual refresh

```bash
~/.config/tmux/calendar/fetch.sh
```

Bypasses the 60s throttle if you delete `~/.cache/tmux-calendar/events.json` first.
